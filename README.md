# NINJA Nav

Vespa Nav is an autonomous 2D mapping robot platform built on the ESP32 architecture. It is designed to navigate indoor environments, detect obstacles via ultrasonic sensors, and generate real-time occupancy grid maps accessible through a custom web interface.

*Figure 1: Prototype hardware platform.*

## Hardware Components

* **Microcontroller:** ESP32 (Dual-Core).
* **IMU:** MPU6050 for orientation tracking and heading stabilization.
* **Proximity Sensing:** 4x HCSR-04 Ultrasonic sensors for obstacle detection.
* **Actuation:** Vespa motor driver platform with differential drive kinematics.
* **Power:** 2x Battery Li-Ion 18650 3,7V 2550mAh.

## Firmware Architecture

The firmware utilizes FreeRTOS to manage hardware tasks and system stability:

* **TaskMapeamento:** It triggers ultrasonic sensors using a dynamic timeout based, applies median filtering, and projects obstacles onto a 100x100 grid matrix using trigonometric calculations. Each pixel i equal to 5cm, so in a 100x100 maping area is equal to 25m²
* **TaskMPU:** Operates on Core 0. Handles I2C communication with the MPU6050. It implements a complementary filter to merge accelerometer and gyroscope data, providing stable yaw values for heading compensation.
* **WebSocket Server:** Uses `ESPAsyncWebServer` for asynchronous communication. It handles joystick input for teleoperation, configuration parameters (calibration), and streams the occupancy grid to the client.

## Description files

* **`ROBO_MAPEADOR.ino`**: Acts as the main system entry point, handling the Wi-Fi/WebSocket server and managing the system's state machine.
* **`variaveis.h`**: Serves as the central configuration header, defining global constants, the occupancy grid matrix, and calibration parameters.
* **`pagina_web.h`**: Contains the full frontend dashboard, integrating HTML, CSS, and JavaScript for user interaction.
* **`mpu_controle.h`**: Processes MPU6050 sensor data using a complementary filter to determine the robot's precise orientation.
* **`movimento_robo.h`**: Translates joystick inputs into differential motor drive commands while calculating real-time odometry.
* **`ultrassom_mapa.h`**: Coordinates the firing and reading of the four ultrasonic sensors while applying median filtering to eliminate noise. It converts raw distance measurements into Cartesian coordinates to update the environment map.

## System Features

* **Real-Time Grid Mapping:** Converts distance data into a 2D occupancy grid with a resolution of 5cm per cell.
* **Low-Latency Teleoperation:** Web-based joystick interface with transmission throttling to prevent buffer overflow and connection drops.
* **Dynamic Calibration:** Allows real-time adjustment of odometry, filtering, and sensing ranges via hidden developer settings.
* **Safety Protocols:** Emergency state machine implemented to halt motors immediately upon Wi-Fi disconnection or critical battery voltage.

## Installation and Setup

1. Configure the ESP32 development environment (Arduino IDE or PlatformIO).
2. Install the following dependencies: `ESPAsyncWebServer`, `AsyncTCP`, and `ArduinoJson`.
3. Upload the code to the ESP32. The robot will initialize its own Access Point (SSID: `Vespa-XXXXX`).
4. Connect to the network and navigate to the robot's IP address (`192.168.4.1`) in a browser.
